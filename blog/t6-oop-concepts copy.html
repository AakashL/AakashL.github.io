Variable Scope in Ruby

September 4, 2015

In programming a variable's scope refers to in which parts of the program the variable's value is accessible. Now that we've covered classes, we've seen all of the difference scopes of variables. In Ruby there are four different variable scopes, and each type is declared differently according to the formatting below:

Name Begins With Scope
$ Global
@ Instance
[a-z] or _ Local
[A-Z] Constant
@@ /td> Class

Usage Examples

Local Variables

Local variables are locally reference-able within the construct they were created in. An example of this is inside and outside methods. Local variables defined inside a method cannot be modified or read outside the method, and the reverse for local variables defined outside of a method.

Global Variables

Global variables are accessible anywhere within the program. On top of this, they are also modifiable from anywhere within the program too. w3schools: "You can set it anywhere, you can get it anywhere".

Instance Variables

Instance variables are local to specific instances of an object. Each instance of a particular object can have it's own value for an instance variable defined in that class.

Class Variables

Unlike instance variables, a class variable is shared amongst all instances of a particular class. If one object in the class changes the value of a class variable, it's changed for all instances of the class.
